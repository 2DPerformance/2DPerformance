Security statement

นายช่างใหญ่ Civil Apps Security Hardening

หน้านี้สรุปมาตรการป้องกันของ public static preview สำหรับผู้ใช้และผู้ทดสอบระบบ

Current protection

  • Production build ไม่เผยแพร่ source maps
  • Deploy มี security check เพื่อกัน secret/source map หลุดขึ้น public
  • หน้าเว็บมี Content Security Policy เพื่อลดความเสี่ยง XSS และ data exfiltration
  • มี runtime guard ล้าง query ที่เป็น token/secret และพยายามกันการฝังเว็บใน iframe ที่ไม่ได้ตั้งใจ
  • หน้า Dashboard มีปุ่มตรวจและล้าง Project/Autosave/Backup ที่ค้างใน browser เครื่องนี้
  • License server config และ license data ถูกแยกจาก static deploy และไม่ควร commit เข้า Git

Important limitation

เว็บที่รันใน browser ไม่สามารถป้องกันการ reverse engineer ได้ 100% เพราะ JavaScript ที่ส่งให้ browser สามารถถูก inspect หรือคัดลอกได้เสมอ หากต้องการป้องกันสูตรคำนวณหรือ license แบบจริงจัง ต้องย้ายส่วนสำคัญไปทำงานบน backend และตรวจสิทธิ์ที่ server

Recommended production path

  • ใช้ backend license validation และ short-lived signed session
  • ย้าย calculation/report signing ที่เป็นทรัพย์สินหลักไป server-side
  • เพิ่ม audit log สำหรับ report export, CAD export, และ license failure
  • ใช้ rate limit และ abuse detection ที่ API edge
